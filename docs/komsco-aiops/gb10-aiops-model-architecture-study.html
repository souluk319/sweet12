Executive Summary

권장 기본안은 역할 분리형 혼합 구성입니다. GB10 #1은 Qwen3.6 27B 또는 Mistral Small 4 계열을 Tool Plan/RCA reasoning 전용으로 두고, GB10 #2는 Gemma 4 26B A4B를 빠른 한국어 운영 질의, 요약, 최종 보고서 정리에 사용합니다.

운영 안정성과 HA가 최우선이면 Gemma 4 26B A4B 동일 모델 Active-Active로 먼저 시작하는 것이 더 현실적입니다. 복합 RCA는 별도 reasoning lane으로 승격시키면 됩니다.
원문 PDF의 tok/s와 별점은 사전 아키텍처 가설입니다. 본 문서는 GB10 2노드 실측 없이 성능 점수를 확정하지 않습니다.

Decision

추천 기본 경로

1안 역할 분리형을 목표 아키텍처로 둔다. 복합 RCA와 Tool Plan 정확도가 사업 핵심이기 때문이다.

운영 시작 경로

2안 Gemma Active-Active로 Gateway, RBAC, RAG, Tool Adapter를 먼저 안정화할 수 있다.

검증 제한 경로

4안 대형모델 결합은 PoC/벤치마크 트랙으로 분리한다. 주력 서비스 선정표에 섞지 않는다.

RAG Store Recommendation

원문 PDF의 PostgreSQL + pgvector 기본안은 Dell Pro Max GB10 단일 장비 또는 소규모 2노드 구성에서 타당합니다. 문서 chunk, embedding, ACL metadata, evidence metadata를 한 DB에 두고 AI Gateway에서 권한 필터와 감사 로그를 통제하는 방식이 운영 부담이 가장 낮습니다.

저장소권장 수준사유
PostgreSQL + pgvector기본안SQL, metadata, ACL, vector를 함께 관리하기 쉬움
Qdrant성능 병목 시 대안전용 vector 검색 성능이 필요할 때 검토
OpenSearch기존 클러스터 있을 때BM25 + vector hybrid가 필요할 때 유리
Milvus대규모 전용소규모 GB10 배치에는 운영 비용이 큼

Final Recommendation Text

KOMSCO AIOps Agentic Model은 OpenShift Lightspeed를 대체하는 단순 챗봇이 아니라, 사내 증적과 운영 도구를 안전하게 연결하는 AI Gateway 중심 운영 분석 계층으로 설계해야 합니다. GB10 2노드 환경에서는 동일 모델 Active-Active로 운영 기반을 먼저 안정화하고, 복합 RCA와 Tool Plan이 필요한 요청만 Qwen/Mistral 계열 reasoning lane으로 승격하는 하이브리드 구성이 가장 현실적입니다.
