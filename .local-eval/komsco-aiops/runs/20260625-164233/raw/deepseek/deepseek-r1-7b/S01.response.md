

**OpenShift 클러스터에서 특정 namespace의 Pod가 반복적으로 CrashLoopBackOff 상태인 경우의 대응 방법**

**1.Pod의 자원 할당 상황을 확인하세요.**
   - `oc get pods -n <namespace> -w | jq '.[] |= { pod: .status, resources: . }'`
   - Pod의 CPU, 메모리,_STORAGE 등을检查看是否足够。

**2.ingress controller(IC)의 네트워크를 Check하세요.**
   - `oc get pods -n <namespace> -w | jq '.[] |= { ingress: .ingress }'` 
   - IC의 Health를检查看 state.
   - `oc get pods -n <namespace> -w | jq '.[] |= { ingress: .ingress }'` 
   - Pod의 INgress policy를检查看是否正确。

**3.네트워크 통路를 Check하세요.**
   - `netcat <pod IP> 80`
   - `ping <pod IP>` 
   - Pod의 네트워크 인터페이스가通畅한지检查看.

**4.Pod의 RESTART policy를检查看.**
   - `oc get pods -n <namespace> -w | jq '.[] |= { pod: .status, restartPolicy: . }'`
   - Pod의 RESTART policy를 Validate.

**5.네트워크 통로를 Check하세요.**
   - `netcat <pod IP> 80` 
   - `ping <pod IP>` 
   - Pod의 네트워크 인터페이스가通畅한지检查看.
