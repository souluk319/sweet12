

### CPU 사용률 급증 원인 조사

#### 1. RHEL (CentOS)сер버
- **목적**: CPU 사용량이 급증하는 원인을 파악하기 위해 Processes를 확인하고 CPU 사용량이 높은 Processes를 찾는다.
- **명령**:
  - `ps -ef | grep -i "cpu"`
    - `ps`는 Processes List를 보여주며, `grep -i "cpu"`는 CPUs가 높은 Processes를 찾는다.
  - `top`
    - `top`는 Current System-wide CPU, RAM, Swap usages를 보여준다. Interval을 5분, 15분, 30분으로 설정한다.
  - `top -n 2 | grep -i "RHEL"`
    - `top`를 2개의窗口로 나누고, RHEL进程에 집중하여 CPU 사용량을 파악한다.
  - `ps aux | grep -i "RHEL"`
    - Processes List에서 RHEL进程만을 선정하여 CPU 사용량을 확인한다.
  - `sudo su -r | grep -i "cpu"`
    - root shell을Spawn하고, CPUs가 높은 Processes를 찾는다.

- **risks/ heavyweight commands**:
  - `sudo su -r | grep -i "cpu"`: root shell을Spawn하면 System-wide Processes에 액세스할 수 있다. System-wide Processes를 manipولировать risk.
  - `ps aux | grep -i "RHEL"`: RHEL进程에 집중하지만, Processes List에서 System-wide Processes除外 risk.

#### 2. Windows Server
- **목적**: CPU 사용량이 급증하는 원인을 파악하기 위해 Services, CPU- Demanding Processes, SQL Server usages를 확인한다.
- **명령**:
  - `services.msc` or `cmd /c "services /all"`
    - Services List를 보여주며, CPU Demanding Services를 찾는다.
  - `top`
    - `top`는 Current System-wide CPU, RAM, Swap usages를 보여준다. Interval을 5분, 15분, 30분으로 설정한다.
  - `netsh winserver query -v service /show name,version,processes,costs,domain`
    - CPU Demanding Services의 CPU usage를 파악하기 위해 network services를 보여준다.
  - `cmdlet Get-ServiceDetails -Name "SqlServer" | Select-Object -Property Value,CPU`
    - SQL Server의 CPU usage를 파악하기 위해 cmdlets를 사용한다.
  - `wmic process /where name contains 'SQL' | select performance,cpupercentage,nice,pid`

- **risks/ heavyweight commands**:
  - `netsh winserver query -v service /show name,version,processes,costs,domain`: SQL Server Demanding Processes를 보여주며, sensitive data exposure risk.
  - `cmdlet Get-ServiceDetails -Name "SqlServer" | Select-Object -Property Value,CPU`: SQL Server의 CPU usage를 파악하는 데 System Services API risk.
