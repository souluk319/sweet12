| 환경 | 확인 명령 | 이유 |
| --- | --- | --- |
| RHEL 서버 | top | 시스템 전체의 CPU 사용률과 각 프로세스의 CPU 사용률을 보여줍니다. |
| RHEL 서버 | vmstat 1 | 메모리, CPU, I/O, 네트워크 사용량을 실시간으로 모니터링합니다. |
| RHEL 서버 | iostat -x 1 | 디스크 I/O 성능과 관련된 상세 정보를 제공합니다. |
| RHEL 서버 | netstat -anp | 네트워크 연결 상태와 각 프로세스의 네트워크 사용량을 보여줍니다. |
| RHEL 서버 | sar -u 1 | CPU 사용률과 관련된 다양한 시스템 통계를 수집합니다. |

| 환경 | 확인 명령 | 이유 |
| --- | --- | --- |
| Windows Server | tasklist /fi "IMAGENAME eq *.*" | 실행 중인 모든 프로세스와 각 프로세스의 CPU 사용률을 보여줍니다. |
| Windows Server | perfmon | 시스템 전체의 성능 데이터를 모니터링하고, CPU 사용률과 관련된 그래픽을 제공합니다. |
| Windows Server | netstat -an | 네트워크 연결 상태를 보여줍니다. |
| Windows Server | iostat /y | 디스크 I/O 성능을 모니터링합니다. |
| Windows Server | systeminfo | 시스템 정보와 관련된 데이터를 수집합니다. |

| 환경 | 확인 명령 | 이유 |
| --- | --- | --- |
| OpenShift Pod | kubectl top pod <pod-name> -n <namespace> | Pod의 CPU 사용률을 보여줍니다. |
| OpenShift Pod | kubectl describe pod <pod-name> -n <namespace> | Pod의 상세 정보와 관련된 데이터를 수집합니다. |
| OpenShift Pod | kubectl logs <pod-name> -n <namespace> | Pod의 로그를 확인하여 에러나 경고 메시지를 찾아냅니다. |
| OpenShift Pod | oc adm top node <node-name> | 노드의 CPU 사용률을 보여줍니다. |
| OpenShift Pod | oc describe node <node-name> | 노드의 상세 정보와 관련된 데이터를 수집합니다. |

위험하거나 부하를 유발할 수 있는 명령은 다음과 같습니다.

| 환경 | 확인 명령 | 이유 |
| --- | --- | --- |
| RHEL 서버 | top -d 0.1 | 너무 빠른 간격으로 실행하면 시스템 리소스 사용량이 증가할 수 있습니다. |
| Windows Server | tasklist /fi "IMAGENAME eq *.*" | 모든 프로세스를 나열하면 메모리 사용량이 증가할 수 있습니다. |
| OpenShift Pod | kubectl top pod <pod-name> -n <namespace> | 너무 자주 실행하면 API 서버의 부하가 증가할 수 있습니다. |

위 명령들은 각 환경에서 CPU 사용률을 확인하고, 문제가 발생했을 때 원인을 파악하는 데 도움이 될 것입니다.
