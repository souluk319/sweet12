### RCA 보고서

#### 1. 증상
`payment-api` Pod이 `CrashLoopBackOff` 상태로 있으며, PostgreSQL 연결 실패와 관련된 오류 로그가 출력되었습니다.

#### 2. 원인 분석
- **증거**: `oc logs payment-api-7c9c7d9f6f-8k2p1 -n payment --previous`에서 "password authentication failed for user 'payment_app'"라는 메시지가 있습니다.
- **추정**: PostgreSQL에 대한 비밀번호 인증이 실패하여 Pod이 재시도 중입니다.

#### 3. 추가 확인 명령
- `oc get secret payment-db-secret -n payment -o yaml`: 비밀번호를 확인합니다.
- `kubectl exec -it <postgres-pod> -n payment -- psql -U postgres`: PostgreSQL에 직접 연결하여 사용자 권한을 확인합니다.

#### 4. 즉시 조치
- 비밀번호를 확인하고 변경해야 합니다.
- 비밀번호 변경 후, Pod의 재시작을 시도합니다.

```bash
# 비밀번호 변경 명령 예시
kubectl get secret payment-db-secret -n payment -o jsonpath='{.data.password}' | base64 --decode

# 비밀번호 변경 후 Pod 재시작
oc rollout restart deployment/payment-api -n payment
```

#### 5. 재발 방지책
- **비밀번호 관리**: 비밀번호를 자주 변경하고, 복잡한 패스워드를 사용합니다.
- **환경 변수 보안**: 중요한 환경 변수는 Kubernetes Secret으로 관리하며, Pod에서 직접 노출되지 않도록 합니다.
- **로그 모니터링**: 중요한 로그 메시지를 모니터링하여 이상이 발생할 경우 즉시 알림을 받습니다.
